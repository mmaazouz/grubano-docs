# Menu & Scan IA

Mettez en ligne votre carte en photographiant simplement vos plats : l’IA reconnaît chaque recette, propose nom, description et allergènes, et vous n’avez plus qu’à valider.

<FlowDiagram steps={["Scanner la carte","Vérifier les plats","Publier","Mettre à jour"]} />

## L’essentiel

Le scanner IA transforme une photo de menu en fiche plat structurée — nom, description appétissante, allergènes, calories estimées, catégorie — en quelques secondes. Vous vérifiez, ajustez le prix et publiez. Idéal pour accélérer la mise en ligne d’une nouvelle carte ou l’ajout ponctuel d’un plat du jour, sans saisie manuelle fastidieuse.

## Comment ça marche

Photographier un menu papier pour le retaper ligne par ligne est chronophage et source d’erreurs. Le scanner IA lit l’image, identifie chaque plat visible et génère automatiquement une fiche complète en français : nom commercial, pitch de deux phrases, liste d’ingrédients, allergènes standardisés (Gluten, Lactose, Œuf, Soja…), fourchette calorique, catégorie (Entrées, Plats, Desserts, Boissons) et labels suggérés (Végétarien, Halal, Sans gluten, Épicé). Vous recevez un JSON prêt à l’emploi que le [tableau de bord](/fr/guides/restaurant/) intègre directement dans votre menu : il ne reste qu’à fixer le prix et valider. La reconnaissance tourne sur un modèle Claude Sonnet 4.5 entraîné sur des cartes de restaurants — elle repère même les plats manuscrits ou dans une mise en page dense.

<Callout type="info">
L’IA propose aussi des variantes (taille Petite/Moyenne/Grande, suppléments Fromage/Bacon…) et une note de préparation si elle détecte une mention spéciale sur la carte. Ces options sont facultatives : vous les activez ou les retirez selon votre offre réelle.
</Callout>

## Étape par étape

<Steps>

### Accéder au scanner

Dans le [tableau de bord](/fr/guides/restaurant/), section **Menu**, cliquez sur l’onglet **À adopter** puis **Scanner une carte** (icône appareil photo). Autorisez l’accès à la caméra ou sélectionnez une image déjà prise.

### Photographier le plat

Cadrez le plat ou la section de menu qui le décrit. Un éclairage naturel et un cadrage droit améliorent la reconnaissance. Validez la prise.

### Vérifier la fiche générée

L’IA renvoie en ~3 secondes un aperçu structuré : nom, description, ingrédients, allergènes, calories min–max, catégorie, labels. Relisez chaque champ — l’IA est fiable mais peut confondre un ingrédient rare ou mal orthographié.

### Ajuster et publier

Éditez le nom ou la description si nécessaire, saisissez le prix de vente (l’IA ne lit pas les montants pour éviter les erreurs de devise), cochez **Disponible**, puis **Enregistrer**. Le plat apparaît immédiatement sur votre carte publique dans la catégorie détectée.

</Steps>

## Exemple concret

Vous ouvrez un comptoir de burgers et souhaitez mettre en ligne dix recettes signature. Plutôt que de remplir dix formulaires à la main, vous photographiez votre carte imprimée section par section. Pour le **Cheese Deluxe**, l’IA propose : « Burger double steak haché 180 g, cheddar fondu, cornichons croquants, sauce maison sur pain brioché. » Elle identifie les allergènes Gluten, Lactose, Œuf (sauce) et Soja (pain), estime 650–750 kcal, classe le plat en **Plats** et suggère le label **Halal** si votre viande l’est. Vous confirmez, ajoutez le prix 12,90 € et trois options Petite/Moyenne/Grande. En dix minutes les dix burgers sont en ligne, contre une heure de saisie manuelle.

## Bonnes pratiques

- **Photographiez plat par plat** si votre carte est dense : l’IA traite mieux une recette isolée qu’une page complète avec quinze entrées.
- **Éclairage indirect** : évitez les reflets sur un menu plastifié qui masquent le texte.
- **Vérifiez les allergènes** : l’IA se base sur les ingrédients visibles ; un condiment caché (moutarde dans une vinaigrette) peut lui échapper. Ajoutez manuellement tout allergène manquant.
- **Fixez le prix vous-même** : l’IA ne lit jamais les montants pour éviter toute confusion entre prix TTC, devise ou promotion. C’est le seul champ obligatoire à saisir après le scan.

<Callout type="warning">
Le scanner IA consomme des crédits API par photo analysée. Une limite raisonnable (quelques dizaines de scans par jour) s’applique par défaut pour préserver les ressources partagées. Si vous atteignez la limite, réessayez quelques heures plus tard ou contactez le support pour un quota élargi.
</Callout>

## Erreurs fréquentes

**« L’IA ne reconnaît pas mon plat manuscrit. »** — L’écriture doit rester lisible ; une calligraphie très stylisée ou un texte trop petit réduisent la précision. Prenez la photo de plus près ou retapez le nom manuellement, puis laissez l’IA générer la description à partir de ce nom.

**« Les calories proposées semblent fausses. »** — L’IA donne une fourchette large (ex. 650–750 kcal) basée sur les portions standard du type de plat. Ajustez la valeur si vous connaissez le poids exact de vos portions ou laissez vide : l’affichage calorique est facultatif côté client.

**« Le plat apparaît dans la mauvaise catégorie. »** — Changez manuellement la catégorie (Entrées, Plats, Desserts, Boissons) avant de publier. L’IA devine d’après le nom et les ingrédients, mais un « Carpaccio de thon » peut hésiter entre Entrées et Plats selon le contexte.

## Astuces & conseils

- **Ré-utilisez les fiches** : une fois scannée, une recette reste modifiable dans votre menu. Changez le prix ou la disponibilité sans re-scanner.
- **Scannez les plats du jour** : photographiez l’ardoise du midi, validez les trois suggestions, publiez en deux minutes. Le soir, désactivez-les d’un clic dans [Gestion des stocks](/fr/guides/stocks/) ou le tableau de bord.
- **Enrichissez la description** : l’IA produit deux phrases génériques ; ajoutez un détail signature (« viande maturée 21 jours », « pâte faite maison ») pour vous démarquer.
