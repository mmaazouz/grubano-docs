# Réservations

Organisez votre plan de salle, acceptez ou refusez les demandes, et suivez l'occupation de vos tables en temps réel.

<FlowDiagram steps={["Configurer la salle","Recevoir la demande","Valider","Service"]} />

## Ce que ça fait

Le module Réservations vous permet de gérer l'ensemble du cycle d'une réservation, de la demande initiale jusqu'à l'arrivée du client. Vous définissez d'abord votre plan de salle — nombre de tables, capacité de chacune, disposition — puis vous recevez les demandes de réservation, que vous pouvez accepter ou refuser selon la disponibilité. Le système vérifie automatiquement qu'une table n'est pas déjà réservée sur le créneau choisi et que le nombre de couverts demandé ne dépasse pas la capacité de la table.

Chaque réservation peut être suivie depuis son statut initial « confirmé » jusqu'à « arrivé » (lorsque le client se présente), « annulé » ou « no-show » (absence sans prévenir). Vous pouvez noter les allergies, une précommande, et ajouter des remarques pour le service. Lorsque vous marquez une réservation « arrivé », le [tableau de bord](/fr/guides/restaurant/) ouvre automatiquement l'addition liée à cette table, ce qui permet d'enchaîner immédiatement la prise de commande.

## Comment l'utiliser

<Steps>

### Créez votre plan de salle

Ajoutez une ou plusieurs tables dans la section « Salle » : indiquez le nom (par exemple « Table 3 »), le nombre de places assises, et leur position sur le plan. Une table inactive n'apparaît plus dans le formulaire de réservation.

### Enregistrez une nouvelle réservation

Sélectionnez la table, la date, l'heure de début, le nombre de couverts, et les coordonnées du client (nom obligatoire, téléphone et email facultatifs). Le système propose automatiquement une durée par défaut (60 minutes, ou la durée configurée pour votre établissement) ; vous pouvez l'ajuster manuellement si nécessaire. Si la table est déjà occupée sur ce créneau, vous recevez un message d'erreur avec les horaires du conflit.

### Confirmez ou annulez la réservation

Le statut initial est « confirmé ». Vous pouvez passer la réservation en « annulé » si le client se désiste, ou en « no-show » s'il ne se présente pas. Les réservations annulées ou no-show libèrent automatiquement la table pour d'autres créneaux.

### Marquez l'arrivée du client

Lorsque le client se présente, passez la réservation au statut « arrivé ». L'addition de service s'ouvre alors automatiquement sur cette table, vous permettant d'enchaîner directement la prise de commande.

</Steps>

## Bon à savoir

<Callout type="info">
Une réservation dont l'horaire de début est déjà passé (de plus de cinq minutes) ne peut plus être créée — le système bloque la demande pour éviter les erreurs de saisie. Si vous devez enregistrer une arrivée sans rendez-vous, créez d'abord une réservation pour un créneau à venir, puis marquez-la immédiatement « arrivé ».
</Callout>
