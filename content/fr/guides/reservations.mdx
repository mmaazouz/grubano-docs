## Le principe (et le pourquoi)

Grubano vous donne une vue unifiée de votre salle et de vos créneaux. Chaque table définie dans le plan de salle peut recevoir une ou plusieurs réservations dans la journée ; le système vérifie automatiquement qu’aucun créneau ne se chevauche et que le nombre de couverts ne dépasse pas la capacité de la table. Vous voyez en un coup d'œil les arrivées prévues, les tables occupées et les slots encore libres.

Lorsque vous marquez une réservation comme « arrivée », le [tableau de bord](/fr/guides/restaurant/) ouvre automatiquement l’addition liée à cette table : le client peut alors commander depuis son téléphone (si vous avez activé le service en salle) ou régler directement quand le repas est terminé. L’empreinte bancaire éventuellement capturée lors de la réservation reste active jusqu’au paiement complet — elle protège contre les no-shows et les additions impayées.

## Suivre vos réservations

<Steps>

### Consultez l’occupation du jour

Dans votre tableau de bord, sélectionnez une date : la liste des réservations confirmées s’affiche, classée par horaire de début. Chaque ligne indique le nom du client, le nombre de couverts, la table attribuée et le statut (confirmée, arrivée, annulée, no-show).

### Créez une nouvelle réservation

Cliquez sur « Nouvelle réservation », choisissez la table, l’heure de début et le nombre de couverts. Le système vérifie que la table est libre sur le créneau et que sa capacité suffit ; si une autre réservation chevauche le slot, un message d’erreur vous prévient. Vous pouvez saisir des allergies, des notes de service ou une précommande.

### Validez ou refusez une demande

Lorsque le créneau vous convient, laissez le statut « confirmée » ; si vous devez annuler (fermeture exceptionnelle, erreur de planning), passez le statut à « annulée ». Le client en est informé par e-mail si son adresse figure dans la réservation.

### Marquez l’arrivée

Dès que le client s’installe, changez le statut en « arrivée ». L’addition s’ouvre automatiquement sur la table : le service peut commencer, les commandes s’ajoutent au ticket en temps réel.

### Clôturez ou traitez un no-show

Si le client règle et part, l’addition passe en « payée » et libère la table. S’il ne s’est jamais présenté, marquez le statut « no-show » : la pénalité éventuelle (définie lors de la réservation) est capturée sur l’empreinte bancaire, et la table redevient disponible pour le créneau suivant.

</Steps>

## Un samedi soir à 20 h

Vous avez une table de quatre couverts réservée pour 20 h, avec une empreinte de 40 € (10 € par personne). Le client arrive à l’heure : vous marquez « arrivée », l’addition s’ouvre, il commande depuis l’application et règle 85 € en fin de repas. L’empreinte est libérée automatiquement au paiement — aucune somme n’est débitée.

Si en revanche le client ne vient pas et n’a pas annulé dans le délai autorisé, vous marquez « no-show » : la pénalité de 40 € est capturée sur l’empreinte, et la table est libérée pour une éventuelle réservation de dernière minute.

Cet exemple est illustratif ; les montants d’empreinte et de pénalité sont définis par vos propres règles de réservation.

## Optimiser la gestion de salle

<Callout type="default">
- **Mettez à jour le plan en temps réel** : si une table est déplacée ou hors service, modifiez sa configuration dans le tableau de bord pour éviter les conflits de réservation.
- **Notez les allergies et préférences** : le champ « notes » est synchronisé avec l’addition, votre équipe de cuisine voit l’information sans manipulation supplémentaire.
- **Marquez l’arrivée dès l’installation** : l’addition s’ouvre instantanément, le client peut commander ou consulter la carte sans attendre.
- **Surveillez les dépassements** : le statut « dépassement » s’active automatiquement si le créneau se prolonge au-delà de l’heure de fin prévue — utile pour anticiper le retournement de table.
</Callout>

<Eyebrow icon="error">Erreurs à éviter</Eyebrow>

## Les pièges courants

<Callout type="error">
- **Ne créez pas de réservation sur une table inexistante** : le système bloque la création si l’identifiant de table est vide ou invalide ; configurez d’abord votre plan de salle.
- **N’acceptez pas plus de couverts que la capacité** : la validation serveur refuse toute réservation qui dépasse le nombre de places de la table sélectionnée.
- **N’oubliez pas de marquer le no-show** : si vous laissez le statut « confirmée » alors que le client n’est jamais venu, la table reste bloquée et l’empreinte n’est pas capturée.
</Callout>
